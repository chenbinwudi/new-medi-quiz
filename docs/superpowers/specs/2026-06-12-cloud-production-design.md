# 医考通云端上线版设计

日期：2026-06-12

## 目标

把当前医考通小程序从本地 MVP 升级为可上线的云端学习产品。小程序需要支持游客使用、登录同步、云端题库、真实学习记录、错题收藏笔记、学习报告、资料中心，以及会员/订单/下载等可点击页面入口。

题库采用原创仿真题，不使用未授权的官方原卷或历年真题原文。题目按执业医师考试常见知识点生成，包含题干、选项、答案、解析、章节、题型、难度和考点标签。医学内容上线前需要人工复核。

## 范围

第一期包含：

- 微信小程序游客模式与登录同步。
- CloudBase 数据库集合与云函数。
- 原创仿真题库种子数据与入库脚本。
- 首页、题库、练习、错题、收藏、笔记、资料、报告、我的页面真实数据接入。
- 会员中心、订单、下载管理等入口页面与状态展示。
- 小程序端功能验收脚本。

第一期不包含：

- 微信支付真实交易。
- 未授权真题原文导入。
- 真实 PDF 文件下载和版权资料分发。
- 后台管理 Web 系统。

## 数据模型

云数据库集合：

- `users`：用户档案，包含 `openid`、昵称、头像、登录时间、会员状态、同步状态。
- `question_categories`：考试、科目、章节、子章节和排序。
- `questions`：原创仿真题，包含题型、题干、选项、答案、解析、章节、难度、标签、状态。
- `papers`：模拟卷、套卷和题目 ID 列表。
- `materials`：资料元数据，包含分类、标题、类型、大小、学习人数、收藏数、状态。
- `answer_records`：答题记录，按用户和题目记录答案、是否正确、用时、来源。
- `wrong_questions`：错题记录，按用户和题目去重，可记录错误次数和最近错误时间。
- `favorites`：收藏记录，支持题目和资料两类收藏。
- `notes`：用户笔记，关联题目、资料或章节。
- `study_reports`：学习报告汇总，包含做题量、正确率、趋势、题型分布。
- `memberships`：会员权益状态，第一期由后台字段控制。
- `orders`：订单占位数据，第一期只展示记录和状态，不接支付。
- `downloads`：下载管理占位数据，第一期只展示资料下载状态。

所有用户私有集合都必须带 `openid`。题库和资料集合使用公开读取、后台维护的策略。

## 登录与同步

小程序默认允许游客使用。游客答题、收藏、笔记和错题先写入本地缓存。

用户登录时调用云函数获取 `openid`，创建或更新 `users` 记录，然后上传本地缓存。云端按以下键去重合并：

- 答题记录：`openid + questionId + source + answeredAt`。
- 错题：`openid + questionId`。
- 收藏：`openid + targetType + targetId`。
- 笔记：本地笔记生成临时 ID，登录后换成云端 ID。

同步完成后重新计算 `study_reports`。如果网络或云函数失败，前端保留本地数据并提示稍后同步，不阻断做题。

## 云函数

第一期云函数按业务边界拆分：

- `login`：获取用户身份并创建用户档案。
- `syncGuestData`：上传游客本地记录并合并。
- `getHomeData`：首页考试、统计、推荐资料。
- `getQuestionBank`：章节、套卷、考点列表。
- `getPracticeSession`：按章节、套卷或错题生成练习会话。
- `saveAnswer`：保存答题并更新错题、报告。
- `toggleFavorite`：收藏或取消收藏题目、资料。
- `saveNote`：新增或更新笔记。
- `deleteNote`：删除笔记。
- `getStudyData`：错题、收藏、笔记、报告聚合数据。
- `getMaterials`：资料分类、列表和详情。
- `getProfileData`：我的页面、会员、订单、下载状态。

已有云函数可以复用并扩展；新功能优先保持函数输入输出稳定，前端通过 `services` 层调用。

## 页面闭环

首页：

- 考试选择读取云端考试/科目。
- 搜索支持题库、资料和考点筛选。
- 8 个快捷入口全部跳转到真实页面或对应 Tab。
- 今日练习、推荐资料读取云端统计和资料集合。

题库页：

- 章节练习、历年真题、模拟试卷、考点速记四个 Tab 从云端读取。
- 点击章节或套卷进入练习页。
- 显示总题数、已做数、正确率和更新时间。

练习页：

- 支持单选、多选、不定项。
- 答题后保存记录、生成错题、更新报告。
- 支持收藏、笔记、答题卡、上一题、下一题。
- 完成后进入结果页或报告页。

错题、收藏、笔记：

- 全部读取云端数据，未登录时读取本地缓存。
- 支持按章节和题型筛选。
- 错题可重新练习，收藏可取消，笔记可新增、编辑、删除。

资料：

- 分类、列表、详情从云端读取。
- 资料详情支持收藏、学习、下载状态展示。
- 下载管理页展示状态，不做真实文件下载。

我的：

- 显示登录状态、会员状态、学习入口。
- 订单、下载管理、我的资料、我的笔记、帮助反馈、设置均可点击进入。
- 会员中心展示权益和当前状态，不接真实支付。

## 题库种子数据

第一期生成 300-500 道原创仿真题，覆盖执业医师医学综合常见模块：

- 基础医学综合。
- 医学人文综合。
- 临床医学综合。
- 预防医学综合。
- 中医学基础。
- 中医临床医学。
- 口腔医学综合。
- 公共卫生综合。

每题必须包含：

- 稳定 `questionId`。
- 题型：单选、多选或不定项。
- 题干和选项。
- 正确答案。
- 解析。
- 章节和考点标签。
- 难度。

种子脚本需要可重复执行，按 `questionId` upsert，避免重复入库。

## 错误处理

云函数失败时，前端展示轻提示并保留本地操作。游客数据不得因同步失败丢失。题库和资料加载失败时显示空状态和重试按钮。保存类操作需要防重复点击。

## 测试与验收

验收包括：

- 静态 UI 检查继续通过。
- MVP 学习流检查继续通过。
- 新增云函数输入输出结构测试。
- 种子数据结构校验。
- 游客做题、登录同步、云端读取的端到端模拟测试。
- 每个首页和我的页入口都能跳转到真实页面。

上线前需要在微信开发者工具中完成：

- 云函数部署验证。
- 云数据库集合和权限验证。
- 真机预览登录、同步、做题、收藏、笔记、报告流程。

## 分阶段实施

1. 建立云端数据模型和种子数据脚本。
2. 扩展云函数，统一服务层接口。
3. 改造页面读取云数据，并保留游客本地兜底。
4. 补齐子页面入口和占位商业页面。
5. 增加自动化验收脚本并修复回归。
6. 输出上线部署清单。
