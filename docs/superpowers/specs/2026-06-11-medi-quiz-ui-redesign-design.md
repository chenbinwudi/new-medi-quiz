# 医考通小程序 UI 重做设计

## 背景

当前小程序已经具备 9 个页面和基础交互，但 UI 验收不合格。主要问题是页面视觉不统一、图标粗糙、卡片和列表样式各自为政、间距和字体层级不稳定，整体质感没有达到用户参考图中的医考工具产品标准。

本次重做不改变业务范围和数据流，重点是建立统一视觉规范、生成 SVG 图标体系，并逐页优化布局、间距、组件状态和验收细节。

## 目标

- 建立统一的医疗专业蓝白视觉系统。
- 生成并接入 SVG 图标资产，替换当前数字块和文字型伪图标。
- 统一 9 个页面的页面边距、卡片圆角、阴影、列表行高、按钮、标签、空状态和底部操作栏。
- 保持微信原生小程序技术栈，不引入第三方 UI 框架。
- 最终在微信开发者工具中通过 UI 验收：不白屏、tab 可切换、每页视觉风格一致、SVG 图标清晰可见。

## 视觉方向

采用“医疗专业蓝白风 + 深色会员卡质感”。

主体页面保持白底、医学蓝、轻卡片、清晰数据层级，符合医考学习工具的可信感和效率感。只有“我的”页会员中心使用深色高质感卡片，以增强商业化入口，但不让整套界面变得厚重。

## 通用组件与样式

### 全局样式

`miniprogram/app.wxss` 需要成为视觉系统入口，包含：

- 页面背景和字体。
- 统一卡片、模块标题、辅助文本。
- 主按钮、次按钮、胶囊标签。
- 页面标题、搜索框、列表行、文件图标。
- 安全区底部操作栏。

### 组件

保留并强化现有组件：

- `seg-tabs`：统一 tabs 高度、激活色、底部指示器。
- `empty-state`：统一空状态图标、标题、说明和按钮。
- `stat-card`：统一数据项排版，可用于首页和学习报告。

新增或统一页面内样式：

- `ui-card`
- `ui-list-row`
- `ui-icon-tile`
- `ui-section-header`
- `ui-progress`
- `ui-bottom-bar`

## 页面级设计

### 首页

首页需要接近参考图的信息密度和视觉层级。

- 顶部展示“医考通”品牌、考试类别、搜索入口和考试倒计时。
- banner 使用医学蓝渐变，加入纸张/笔/学习计划 SVG 插画感元素。
- 快捷入口改为 SVG 图标宫格，图标底色统一，文字不换行、不挤压。
- 今日练习卡展示题目数、做题数、正确率，使用统一数据卡。
- 推荐资料列表使用统一文件图标、标题、类型、大小和学习人数。

验收标准：

- 首屏不空、不挤、不出现数字块替代图标。
- 8 个快捷入口图标风格一致。
- banner、统计卡、资料列表视觉统一。

### 题库

- 顶部展示页面标题、科目选择、搜索图标。
- tabs 使用统一 `seg-tabs`。
- 总进度卡包含总体进度百分比、已学题量和进度条。
- 章节列表行统一左侧圆形加号、章节名、中间题量、右侧进度。
- 空状态和非章节 tab 状态有可理解提示。

验收标准：

- 章节列表行高一致，右侧数字对齐。
- 总进度卡和参考图信息结构一致。
- tabs 切换不破坏布局。

### 章节练习

- 页面背景以白色阅读体验为主。
- 顶部包含返回、标题、设置/更多入口的视觉位置。
- 题型标签、题号、题干层级清晰。
- 选项使用圆形字母标识，选中态蓝色，正确态绿色，错误态红色。
- 解析卡包含“正确答案”“您的答案”“解析”。
- 底部固定操作栏包含收藏、笔记、答题卡、上一题、下一题。

验收标准：

- 题干和选项可读性高。
- 未提交、已提交、正确、错误状态明显。
- 底部操作栏不遮挡内容，适配安全区。

### 错题本

- 顶部返回、标题、编辑入口对齐。
- tabs 包含全部错题、按章节、按题型。
- 错题卡展示题型标签、题干、章节、错误次数和日期。
- 底部“错题重做”主按钮固定。

验收标准：

- 错误次数红色突出但不过分刺眼。
- 多张卡片间距一致。
- 空状态可引导去练习。

### 我的收藏

- 结构与错题本保持一致，但强调“收藏”状态。
- 收藏卡展示题型标签、题干、章节和收藏日期。
- 底部“查看收藏”主按钮固定。

验收标准：

- 与错题本视觉一致但语义区分清楚。
- 列表与筛选状态稳定。

### 学习报告

- 顶部返回、标题、tabs。
- 正确率趋势使用轻量折线或点线图，背景卡片统一。
- 做题战绩使用三列数据卡。
- 题型分布使用环形图视觉或分布条，配图例。
- 能力分析 tab 首版展示示例数据并明确为示例。

验收标准：

- 数据模块像一个统一仪表盘。
- 图表不粗糙、不溢出。
- 空数据时仍有美观空状态和去练习入口。

### 资料

- 搜索框使用统一搜索图标和浅灰背景。
- 资料分类使用 SVG 图标宫格，颜色体系统一。
- 推荐资料列表使用文件卡片，左侧文件 SVG，右侧标题和元信息。

验收标准：

- 分类图标不是彩色数字块。
- 列表行信息对齐，学习人数弱化处理。

### 资料详情

- 顶部文件信息卡：文件 SVG、标题、类型、大小、学习人数、收藏。
- 简介/目录 tabs 与题库 tabs 风格一致。
- 简介字段使用两列元信息。
- 目录列表有序号、标题和右侧箭头。
- 底部分享和立即学习按钮统一。

验收标准：

- 文件卡有明确主体感。
- 底部按钮不漂浮混乱。
- 收藏图标状态清晰。

### 我的

- 用户信息区包含头像、昵称、ID 和进入箭头。
- 会员中心使用深色渐变卡，包含权益文案和立即开通按钮。
- 我的学习入口使用 SVG 图标一排展示。
- 菜单列表使用统一行高、左侧操作图标、右侧箭头。

验收标准：

- 会员卡质感高于普通卡片，但不破坏整体蓝白风。
- 菜单行对齐整齐，图标一致。

## 实现约束

- 不改变现有业务数据模型和云函数接口。
- 不引入第三方 UI 库。
- SVG 文件必须是独立文件，不使用截图或位图替代。
- WXML 中避免复杂表达式，继续使用 JS 预计算 class、style、展示文案。
- 不提交微信开发者工具生成的本地 private 配置。

## 验收方法

### 自动检查

- JS/JSON 语法检查通过。
- WXML 中不出现复杂表达式：`? :`、`===`、`&&`、`index + 1`。
- `miniprogram/assets/icons/` 中存在本 spec 列出的 SVG 图标。
- 9 个页面文件存在且引用统一样式或统一图标。

### 微信开发者工具验收

在微信开发者工具中执行清缓存并编译，逐页检查：

- 首页不是白屏，内容首屏完整。
- 4 个底部 tab 可以切换。
- 章节练习可以选择和提交答案。
- 错题本、收藏、报告、资料详情等二级页可以进入。
- SVG 图标清晰显示，无缺图。
- 页面风格统一，卡片、按钮、标签、间距一致。

## 非目标

- 不重做业务功能。
- 不接真实支付。
- 不做 PDF 阅读器。
- 不做后端题库管理。
- 不追求像素级完全复刻参考图，但要达到同等层级的专业感、统一性和完成度。
